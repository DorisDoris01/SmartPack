# SmartPack MVP 需求规格说明 (Spec)

> **文档用途**：记录本次新增/修改的需求，清单 Item 分类与用户选择的标签一一对应。  
> **基于**：产品需求文档 (PRD) 修改版。  
> **状态**：待实现。

---

## 1. 需求概述

- **核心变更**：生成的清单 **Item 分类** 与用户选择的 **标签** 一一对应，清单结构完全由用户输入决定。
- **输入维度**：性别、出行时间、目的地、旅行活动、特定场合、出行配置。
- **输出**：按「基础清单 + 各标签对应分类」生成的打包清单。

---

## 2. 用户输入字段 (Input Fields)

| 维度     | 字段名称   | 输入类型     | 选项/说明 |
|----------|------------|--------------|-----------|
| **基础** | 性别       | 单选         | 男性 (Male)、女性 (Female) |
| **时间** | 出行时间   | 日期选择器   | 用于确定行程天数 |
| **地点** | 目的地     | 城市选择器   | 全球城市选择 |
| **活动** | 旅行活动   | 多选标签     | 跑步、攀岩、潜水、露营、越野、摄影、摩旅、骑行 |
| **场景** | 特定场合   | 多选标签     | 宴会、商务会议、自驾 |
| **配置** | 出行配置   | 多选标签     | 国际旅行、带娃、带宠物 |

---

## 3. 核心物品生成逻辑 (Item Logic)

### 3.1 基础清单 (Base Items)

由 **性别** 驱动，分为共有项与性别特有项。

#### 共有基础项（不限性别）

- **证件**：身份证/护照、少量现金
- **电子**：手机充电线、充电宝
- **药品**：创可贴、止泻药
- **卫生**：纸巾

#### 性别特有项

| 男性 (Male) | 女性 (Female) |
|-------------|----------------|
| 剃须刀、洗面奶、洗发水、面霜、唇膏、发胶、内裤、袜子、睡衣 | 卸妆油、洗面奶、洗发水、护发素、化妆品、面膜、精华、防晒霜、化妆棉、发圈、备用卫生棉/棉条、内裤、袜子、睡衣、内衣 (Bra) |

## 5. 与现有实现的差异（待对齐）

| 项目 | 当前实现 (PRD v1.4) | 本 Spec 目标 |
|------|---------------------|--------------|
| 标签组 | 场景、运动 | 旅行活动、特定场合、出行配置 |
| 标签内容 | 商务会面、正式晚宴、跑步、游泳、瑜伽等 | 跑步、攀岩、潜水、露营、越野、摄影、摩旅、骑行；宴会、商务会议、自驾；国际旅行、带娃、带宠物 |
| 时间输入 | 出行时长（短途/中途/长途/极长途） | 出行时间（日期选择器 → 行程天数） |
| 地点 | 无 | 目的地（城市选择器） |
| 基础清单 | 通用 + 性别过滤 | 共有基础项 + 性别特有项（与本 Spec 一致化） |

---

## 6. 实现建议

1. **数据层**：新增/调整预设数据结构，使标签组、标签及 Item 列表与本 Spec 第 2、3、4 节一致。
2. **新建行程页**：支持日期选择器、目的地（城市）、以及三组多选标签（旅行活动、特定场合、出行配置）。
3. **生成逻辑**：先加「基础清单（按性别）」，再按用户所选标签依次追加对应分类下的 Item，合并去重。
4. **Item 管理**：若保留「用户可在某标签下增删自定义 Item」，需确保标签 ID 与本节分类一致，便于扩展。

---

## 7. 附录：数据模板约定

- 标签组 ID 建议：`activity`（旅行活动）、`occasion`（特定场合）、`config`（出行配置）。
- 标签 ID 建议：英文小写+下划线，如 `running`、`business_meeting`、`with_kids`。
- Item 可沿用「id + name（中/英）」结构，与现有 PRD 数据结构兼容。
