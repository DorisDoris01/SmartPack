Implementation Plan

File Action Changes
Views/Settings/ItemManagementView.swift MODIFY All three fixes live in this single file
1 U1 — Add alert for last-item delete guard
ItemManagementView.swift
  • Add @State private var showDeleteBlockedAlert = false
  • In both deletePresetItem() and deleteCustomItem(), when canDeleteItem() returns false: set the alert flag + call HapticFeedback.warning()
  • Add .alert() modifier to the List with bilingual title "Cannot Delete" / "无法删除" and message explaining at least 1 item is required
2 U2 — Wire haptic feedback on all interactions
ItemManagementView.swift + AddCustomItemSheet
  • Expand/collapse tag (DisclosureGroup set binding): HapticFeedback.light()
  • Delete preset item: HapticFeedback.warning() before calling deletePresetItem()
  • Restore preset item: HapticFeedback.light() before calling restorePresetItem()
  • Delete custom item: HapticFeedback.warning() before calling removeCustomItem()
  • Add item success (in AddCustomItemSheet.addItem()): HapticFeedback.success() before dismiss()
  • Add item failure: HapticFeedback.error() before showing error alert
  • Last-item blocked (from Step 1): already has warning()
3 U3 — Unify delete pattern to swipe-only + add Custom badge
ItemManagementView.swift
  • Remove the visible minus.circle.fill button from customItemRow()
  • Add .swipeActions(edge: .trailing, allowsFullSwipe: false) to custom item rows — same pattern as preset rows
  • Add a "Custom" / "自定义" badge to custom item rows (orange tint: rgba(orange, 0.1) background)
  • Change "Preset" badge from Color(.systemGray5) to AppColors.primary.opacity(0.1) with AppColors.primary text
4 Wrap mutations in withAnimation
ItemManagementView.swift
  • Wrap deletePresetItem(), restorePresetItem(), removeCustomItem() calls inside withAnimation(PremiumAnimation.snappy)
  • This gives smooth row removal/insertion matching the rest of the app
5 Build & verify
  • Run xcodebuild — confirm zero errors
  • Verify: swipe last item → alert appears
  • Verify: all actions trigger correct haptic type
  • Verify: preset and custom items both use swipe-to-delete
  • Verify: badges show "Preset" (teal) and "Custom" (orange)
